日志/2020-09-01

2020-09-01

上午：继续Linux学习

  1. MBR，GPT —— GPT是未来
  2. MBR最大2T，最多四个主分区
  3. mount / umount
  4. 新分好区的磁盘要制定文件系统
  5. /etc/fstab配置开机自动挂载
  6. mkswap创建交换区
  7. inode存储所有文件的元信息，注意唯独不包括文件名！
    • File type - regular file, directory, character device, etc
    • Owner
    • Group
    • Access permissions
    • Timestamps - mtime (time of last file modification), ctime (time of last attribute change), atime (time of last access)
    • Number of hardlinks to the file
    • Size of the file
    • Number of blocks allocated to the file
    • Pointers to the data blocks of the file - most important!
  8. 任何一个目录的”硬链接”总数，总是等于2加上它的子目录总数（含隐藏目录）
  9. 硬链接使用同一个inode，删除一个不影响另一个
  10. 软链接是一个新的文件，类型为l，如果指向的文件被删除就报错了
  11. 硬链接是用inode去找的，软链接是用文件名去找的，所以硬链接无法跨文件系统，软链接可以
  12. MBR把bootloader存在磁盘的第一个扇区，即第一个512字节里。
  13. UEFI把bootloader存在.efi文件里
  14. Linux bootloader: GRUB
  15. initramfs
  16. 三种创建Init进程的方法：System V，Upstart，systemd
  17. 硬件，内核，用户
  18. /etc/modprobe.d文件夹下添加自动加载module
  19. options / blacklist 决定加载还是不加载module

中午：改一下API，筛选器不再筛列; 看操作系统的书第三章

  1. process layout: text, data, heap, stack

    text: The executable code

    data: Global variables

    heap: dynamic allocated memories

    stack: temporary data when invoke function(argument, return address, localvars)

  2. heap and stack grow toward each other, avoid overlap

  3. process state:

  4. PCB: In brief, the PCB simply serves as the repository for all the data needed to start, or restart, a process, along with some accounting data.

  5. Linux上<include/linux/sched.h>定义PCB结构，父子节点使用双链表存储

  8. System V init被systemd取代，systemd是现在的1号进程了

  9. fork, exit, wait

  10. IPC

晚上：继续看操作系统的第三章

  1. 共享内存实现IPC(区分有无容量限制)
  2. 直接通道：对称通道和不对称通道
  3. 间接通道：mailbox(区分归属操作系统还是进程，owner问题）
  4. 阻塞/非阻塞，无容量/容量/无限容量
  5. 跨电脑：sockets和RPC
