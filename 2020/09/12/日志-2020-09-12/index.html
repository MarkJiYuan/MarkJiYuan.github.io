上午：杂务

  1. HTTP与HTTPS的区别

    HTTPS 是SSL + HTTP，由于HTTP是明文传输的，而HTTPS传输的是加密的信息，因此比HTTP更为安全，中间人攻击的成本更高。具体而言，HTTPS的请求流程是：

    1. 客户端请求建立SSL连接
    2. 服务器将证书传送给客户端，证书中含有公钥
    3. 客户端验证证书是否有效
    4. 客户端生成随机数，使用公钥加密形成会话密钥，然后传给服务器
    5. 服务器使用私钥解密会话密钥
    6. 双方使用会话密钥对称加密要传输的信息

  2. HTTPS的弊端

    1. 连接建立过程比较复杂，耗时耗资源
    2. SSL信任链体系并不完善

中午：网易笔试，学习

  1. /var 文件夹存储所有变化的数据，实时的数据，包括log
  2. The syslog service manages and sends logs to the system logger.
  3. /var/log/messages记录非严重的，非debug的日常信息
  4. /var/log/syslog记录除了auth以外的全部日志
  5. /var/log/kern.log记录kernel日志，包括dmesg
  6. /var/log/auth.log记录登陆信息
  7. logrotate管理log，清理，压缩等
  8. java的fork-join Model
  9. OpenMP模式并行
  10. 苹果公司的GCD（Grad Central Dispatch）
  11. 线程并发的五种方式：
    1. 线程池，不需要每次创建开销
    2. Java fork join，传统的fork join的升级版。需要程序员手动将任务拆分在不同的线程执行，然后将结果合并。
    3. OpenMP，声明并行的代码块，编译器自己切分任务
    4. 苹果的GCD模式。维护任务队列，FIFO。任务分为串行任务和并行任务，串行一次取一个，并行一次可以取好几个。将并行任务区分为4种优先级，以保障交互性为最高优先级。
    5. TBB。感觉类似OpenMP，但是更重视整体时间最短，耗时少的先执行。

晚上：

  1. 进程与线程的区别总结

    线程具有许多传统进程所具有的特征，故又称为轻型进程(Light—Weight Process)或进程元；而把传统的进程称为重型进程(Heavy—Weight Process)，它相当于只有一个线程的任务。在引入了线程的操作系统中，通常一个进程都有若干个线程，至少包含一个线程。

    根本区别：进程是操作系统资源分配的基本单位，而线程是处理器任务调度和执行的基本单位

    资源开销：每个进程都有独立的代码和数据空间（程序上下文），程序之间的切换会有较大的开销；线程可以看做轻量级的进程，同一类线程共享代码和数据空间，每个线程都有自己独立的运行栈和程序计数器（PC），线程之间切换的开销小。

    包含关系：如果一个进程内有多个线程，则执行过程不是一条线的，而是多条线（线程）共同完成的；线程是进程的一部分，所以线程也被称为轻权进程或者轻量级进程。

    内存分配：同一进程的线程共享本进程的地址空间和资源，而进程之间的地址空间和资源是相互独立的

    影响关系：一个进程崩溃后，在保护模式下不会对其他进程产生影响，但是一个线程崩溃整个进程都死掉。所以多进程要比多线程健壮。

    执行过程：每个独立的进程有程序运行的入口、顺序执行序列和程序出口。但是线程不能独立执行，必须依存在应用程序中，由应用程序提供多个线程执行控制，两者均可并发执行

  2. 接口的Rt（ResponseTime响应时间）

  3. PV（Page View）：页面访问量，每次用户访问或者刷新页面都会被计算在内。

    UV(Unique visitor) 一天内同个访客多次访问仅计算一个UV。

    VV(Visit View) 打开网页并最终关闭选项卡，VV+1。

    QPS（Query Per Second）：每秒查询数，每秒系统能够处理的查询请求次数。

    TPS（Transactions Per Second）：每秒事务数，每秒系统能够处理的事务次数。

  4. 日PV=QPS*60*60*24 //即QPS乘以一天的秒数

    峰值QPS=(日PV*80%)/(60*60*24*20%）*//通用公式每天80%的访问集中在20%的时间里，这20%时间叫做峰值时间*

  5. Tip：P0—PN 是互联网大厂经常用来判定事故等级的机制，P0是最高等级了。

  6. 别用ActiveMQ和RabbitMQ，用RocketMQ和Kafka

  7. TLS ( thread local storage )

  8. Linux真的强。Linux并没有从根本上区别process和thread，而是通过clone时的flag来决定更像process还是thread。极高的自定化空间。Linus，永远滴神。
