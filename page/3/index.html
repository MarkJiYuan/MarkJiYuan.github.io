Linux/Linux命令大全

Linux命令清单(我目前会的)

Command Explaination
ls List Directories
cd Change Directories
pwd Print Working Directories
touch 创建新文件/更新timestamp
file 文件的类型信息
cat 可以组合多个文件并打印
less 浏览文本，小名字起的真别致
history 终端历史命令
!! 上一条命令
Ctrl + R 匹配历史命令
cp 复制文件或文件夹，-r复制文件夹，-i防止覆盖
mv 移动文件或文件夹
mkdir 创建文件夹，-p可以连续创建
rm Remove，删除，-r删除目录，-f无视写保护
find 找文件或文件夹，-type，-name参数
help I need help！
man 加强版help，manual
whatis 打印manual的description部分
alias/unalias 别名/取消别名
exit/logout 关闭终端
“>” 重定向标准输出(stdout)，redirection
“>>” 不覆盖，而是添加
“<” 重定向标准输入
2> stderr重定向，可以使用2>&1以及简写&>一并重定向
“|” 将上一个Command的stdout当作后面command的stdin
tee 读取stdin，写入stdout，注意区别于重定向
env 操作环境变量，无参数则打印，$KEY获取特定KEY的value
cut cut out selected portions of each line of a file
paste 将多行文件拼成一行，-d设置delimiter，要带上-s
head 某个stdin的头部，-c设置字符数量，-n设置行，默认10
tail 某个stdout的尾部，-n设置行数，-f不立刻返回，等待变化
expand 将TAB转为空格
unexpand 将空格转回TAB
join 将文件水平拼接
split 将文件纵向分割
sort 将stdin排序，-r反序，-n按照数字排序
tr translate，字符集转换，删除指定字符
uniq 去重，-c Count，-u Uniq，-d Duplicate
wc word count，输出分别是词，行，字节数
nl number lines，给文件加行号
grep 匹配符合的行，-i区分大小写
visudo 安全更改/etc/sudoers
chmod 修改文件权限
chown 改文件所属User
chgrp 改文件所属Group
umask 更改文件默认权限，注意数字表示去除的权限
ps Processes
kill 默认发送的是SIGTERM，让进程正常结束，kill -9则是SIGKILL
nice 设置NI值，越低表示分到CPU的优先级越高
renice 更改一个已有进程的NI值，给定PID
sleep 延迟多少秒，是一个job
ctrl + Z stop一个进程
bg 把一个Stop状态的job变成后台运行
jobs 查看所有job
fg 把一个job放到前台。fg，bg，kill job的时候要加%在job ID的前面
gzip 压缩一个文件，后缀.gz
gunzip 解压一个.gz
tar 把多个文件打包
mknod 创建设备
rpm Redhat 安装软件
dpkg Debian 安装软件
yum Redhat 包管理
apt Debian 包管理
make 编译源码为可执行文件
make install 真正安装
mknod make node，创建设备/dev
udevadm udevadm info –query=all –name=/dev/sda
lsusb USB设备
lspci PCI设备
lsscsi SCSI设备
dd 用于拷贝，cp拷贝的是文件，dd则是块，慎用
cal calendar，打印日历
ncal 跟cal类似，不过方向是不一样
df 查看磁盘使用情况，文件系统等，-h是给人看的
parted 磁盘分区，MBR，GPT都支持
fdisk 支持MBR，不支持GPT
gparted GUI版本的parted
gdisk 支持GPT，不支持MBR
mkfs 创建文件系统
blkid 打印设备信息，如UUID
mount 挂载文件系统
umount 取消挂载
mkswap 创建交换区swap
swapon 交换区，启动！
swapoff 交换区，关闭！
du disk usage
fsck file system check
stat 查看文件的元信息
ln 创建硬链接 hardlink
ln -s 创建软链接 symlinks
strace 打印程序的system call和signal
uname 打印系统信息，如uname -r打印内核版本
lsmod 查看内核modules
modprobe 加载module，-r 卸载
size 可以查看可执行文件的各个区域大小(text,data,heap,stack等)
pstree 树状的ps
service 启动System V脚本
Command Explaination
shutdown 关机，-h +2延迟2分钟执行
reboot 重启
top 任务管理器，活动监视器
uptime 一行查看系统运行信息（时长，用户，CPU负载）
lsof list of open files
fuser identify processes using files or sockets
iostat 查看CPU工作，磁盘读写情况统计数据
vmstat 功能相比iostat更强大，可以看系统中断，swap
sar 需要额外安装的强大的分析工具
cron 定时任务，语法需要注意下
logger 可以调用syslog，logger -s Hello
dmesg 操作/var/log/dmesg，boot期间的系统日志
logrotate 清理，压缩log，常常配合cron使用

晚上：

  1. FTP（File Transfer Protocol，文件传输协议）默认情况下FTP协议使用TCP端口中的 20和21这两个端口，其中20用于传输数据，21用于传输控制信息。控制 Socket 用来传送命令，数据 Socket 是用于传

    送数据。

    FTP的传输有两种方式：ASCII、二进制。

  2. 数据交换方式：

    说明 优点 缺点
    电路交换：在使用电路交换进行通话之前，必须先拨号建立一条连接，也就是一条专用的物理通路，这条通路在通话中只允许通话的两个用户使用，而不允许其他人使用。 实现简单 建立大型网络时资源利用率不高，当用户比较多时，可能会出现通路短缺的情况。
    报文是通信链路中一次要发送的数据，报文交换就是把整个报文完整的发送到链路中，在某个节点存储下来之后再发送到下一个节点。 相比电路交换，来说更加灵活。不需要事先建立连接之后再进行通信。 当每个报文的数据量较大时，每次在节点处转发再存储的话时延较大。
    分组交换采用转发存储技术，将一个完整的报文，分成若干个分组，再进行转发，而且每个分组之间经过哪一个节点，与上一个分组完全没有关系，这一点在某些网络节点发生网络阻塞时会显得尤其重要。 发送数据更加灵活，时延更下。 发送设备和接收设备就更加复杂。

  3. 物理层常用设备：中继器、集线器

    数据链路层常用设备：网桥、交换机

    网络层常用设备：路由

    传输层常用设备：网关

  4. OSPF, RIP运行于自治系统之内，BGP运行在自治系统之间。

  5. OSPF 网络层协议 ——直接使用IP数据报发送数据

    RIP 应用层协议——使用UDP传送数据

    BGP 应用层协议——使用TCP传送数据

  6. STP（生成树协议）的原理是按照树的结构来构造网络拓扑，消除网络中的环路，避免由于环路的存在而造成广播风暴问题。

  7. 冲突域（物理层）：
    同一时间内只能有一台设备发送信息的范围。如果有多台设备同时发送信息，就会产生冲突，故称冲突域。

  8. 广播域（数据链路层）：
    广播帧所能到达的所有的设备的集合。

  9. 隔离及扩展：
    1、物理层的设备（如集线器Hub，中继器）可以扩大冲突域，而不能隔离冲突域，更不能隔离广播域。
    2、数据链路层的设备（如交换机）可以隔离冲突域，扩大广播域。
    3、网络层的设备（如路由器）可以隔离广播域。

  10. FTP分为主动模式和被动模式，任何一种模式中客户端均是动态申请一个端口来使用，服务器端控制连接端口（默认21）始终不变，当使用主动模式时，客户端申请一个端口告诉服务器，服务器则使用20号端口向客户端的这个端口主动建立一条数据连接；当使用被动模式时，服务器则动态申请一个端口告诉客户端以等待客户端的连接
