node/sqlite3使用说明

sqlite3使用说明

目前最新版本为5.0.0

  1. 安装

  2. Main

    1. new sqlite3.Database

      filename=”:memory:”只在内存中存储，程序结束释放。

      mode设置读写模式

      callback处理open和error event

    2. sqlite3.verbose()

      输出比较长的错误栈，用于开发时的调试。

  3. Database(假设instance是db)

    1. db.close

    2. db.configure

      设置sql执行回调等

    3. db.run

      sql挖空的话，最好采用如下这种方式：

    4. db.get

      大体类似于db.run，但是callback不太一样，这里callback的形式是function(err, row) {},

      且callback只对第一行结果生效

    5. db.all

      也是callback执行的条件不一样，是对所有结果生效

    6. db.each

      对每一行分别callback

    7. db.exec

      连续执行多条sql，出错就中断

    8. db.prepare

      返回一个Statement对象

    9. db.serialize

      串行模式，可以嵌套

    10. db.parallelize

      默认就是并行模式，这个是为了在串行模式中令某一部分并行使用的。

  4. Statement(instance是stmt)

    1. stmt.bind

      绑定参数

    2. stmt.reset

      重置已绑定的参数和游标

    3. stmt.finalize

      可以finalize某个Statement，之后所有的调用都会抛出错误。

      剩下的都类似上面了。

    4. stmt.run

    5. stmt.get

    6. stmt.all

    7. stmt.each

上午：学习Linux

  1. 学习了Vim

  2. Linux用户管理-UID, GID

  3. sudo(superuser do)

  4. su(substitute user)

  5. visudo命令安全更改/etc/sudoers

  6. /etc/passwd管理用户密码

    username:password:UID:GID:注释(GECOS):根目录:使用的shell

  7. vipw可以修改/etc/passwd，但是最好不用

  8. /etc/shadow存储加密的密码以及各种过期时间

  9. /etc/group管理组

    group name:group password:GID:User list

  10. 管理用户命令

    Command Explaination
    useradd 创建用户/更新已有用户
    userdel 删除用户
    passwd [username] 更改用户密码

  11. 文件权限

    第一个字符表示文件类型，d为目录，dash为普通文件，l为链接，b为块设备，c为字符设备

    r-可读， w-可写，x-可执行，-为空

    第一个组为自己的权限，第二个组为所属Group的权限，第三个为其他用户的权限

  12. chmod

    两种方式：

    1. chmod uga+rwx

      uga表示user，group，another user，+/-号给或取消权限，rwx是要操作的权限

    2. chmod 755

      1为x，2为w，4为r，所以7为rwx，6为rw-，5为r-x，4为r–，3为-wx，2为-w-，1为–x，0为—

  13. chown，chgrp

    sudo chown patty:whales myfile

    可以一行同时改用户和组

  14. umask 021，默认是umask 022

  15. SUID - 4

    sudo chmod u+s myfile

    sudo chmod 4755 myfile

  16. SGID - 2

  17. sticky bit - 1

  3. 配置文件/etc/gitlab/gitlab.rb

  5. gitlab-ctl restart

  6. 生成SSH key

    然后产生好的公钥在~/.ssh/id_rsa.pub

  7. Gitlab使用。merge request，issue，milestones等

下午：写巴士sqlite采集代码，牛客

  1. npm(node package manager)

  2. npm config get/set/delete xxx

  3. npm dependencies 里

    Version: a.b.c

    a - major，可能重写了API

    b - minor，新特性，不影响现有

    c - patch，补丁，修复bug

    ex：
    ^4.17.4 锁定major，minor，patch最新

    ~4.17.4 锁定major，minor，patch最新

    4.17.4 锁定全部，精确版本

    “*” 全部最新

  4. npm root -g

  5. nodemon 发现文件改变并重启服务

    结论：能用let尽量用let，块作用域，更安全

Linux/Vim使用

Vim使用

  1. 使用/pattern进行向后搜索，?pattern向前搜索，回车确认，n为下一个结果，N为上一个结果

  2. h, j, k, l 分别是 左下上右

  3. 插入模式（Insert）

    Command Explaination
    i 在光标处插入
    o 向下插入一空行
    O 向上插入一空行
    a 在光标后插入
    A 在行尾插入

    Esc退出，回到Command模式

  4. 复制粘贴删除

    Command Explaination
    x 删除光标所在字符
    dd 删除一整行
    y 复制
    p 粘贴

  1. 保存与撤回

    Command Explaination
    :w 保存
    :q 退出
    :wq 保存并退出
    :q! 不保存就退出
    ZZ 相当于:wq，但更顺手
    u Ctrl + z
    Ctrl + r redo last action
