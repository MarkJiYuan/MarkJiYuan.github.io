天平找质量不等的小球

这一类问题描述大概都是这样的：

我在字节的面试以及腾讯的笔试都碰到过这个题，n不一样而已，可见这题还是很常见，值得一记的。

不难发现这个问题是可以递归解决的，即问题的最优性可由子问题的最优性得到。

这个问题乍一看披着一个二分的外表，但实际上是三分才对。将一堆球尽可能三等份，把数量一样的那两堆上称，那么轻的那一堆要么在称上，可以看出来是哪一堆，要么是不在称上的那一堆。随后问题就化简了。

找规律：

球数（n） 最多次数
2 1
3 1
4 2
5 2
6 2
7 2
8 2
9 2
10 3
11 3
12 3
… 3
27 3
… 4
81 4
82 5
83 5

结论：

如 27 < 79 <= 81，因此需要4次。

  8. 几种信号的区别：

    • SIGHUP - Hangup, sent to a process when the controlling terminal is closed. For example, if you closed a terminal window that had a process running in it, you would get a SIGHUP signal. So basically you’ve been hung up on
    • SIGINT - Is an interrupt signal, so you can use Ctrl-C and the system will try to gracefully kill the process
    • SIGTERM - Kill the process, but allow it to do some cleanup first
    • SIGKILL - Kill the process, kill it with fire, doesn’t do any cleanup
    • SIGSTOP - Stop/suspend a process

  9. Linux进程状态值：

    • R: running or runnable, it is just waiting for the CPU to process it
    • S: Interruptible sleep, waiting for an event to complete, such as input from the terminal

    • Z: Zombie, we discussed in a previous lesson that zombies are terminated processes that are waiting to have their statuses collected
    • T: Stopped, a process that has been suspended/stopped

Linux/ps命令

ps命令详解

  1. ps aux

    BSD风格

    a表示all，所有进程，无视用户。因为默认ps只打印自己的进程

    u打印详细信息

    x表示也列出没有terminal的进程，即守护进程daemon，它们的tty是??

    l为长信息，long message，有PPID这些

  2. 各种信息说明

    • USER: The effective user (the one whose access we are using)

    • PID: Process ID

    • %CPU: CPU time used divided by the time the process has been running

    • %MEM: Ratio of the process’s resident set size to the physical memory on the machine

    • VSZ: Virtual memory usage of the entire process

    • RSS: Resident set size, the non-swapped physical memory that a task has used

    • TTY: Controlling terminal associated with the process

    • STAT: Process status code

    • START: Start time of the process

    • TIME: Total CPU usage time

    • COMMAND: Name of executable/command

  3. RSS， VSZ， PSS

  4. top命令可以获取实时状态

  5. Linux下显示系统进程的命令ps，最常用的有ps -ef 和ps aux。这两个到底有什么区别呢？两者没太大差别，讨论这个问题，要追溯到Unix系统中的两种风格，System Ｖ风格和BSD 风格，ps aux最初用到BSD中，而ps -ef被用在System V Style中，两者输出略有不同。现在的大部分Linux系统都是可以同时使用这两种方式的。

  6. System V风格

    UID //用户ID、但输出的是用户名
    PID //进程的ID
    PPID //父进程ID
    C //进程占用CPU的百分比
    STIME //进程启动到现在的时间
    TTY //该进程在那个终端上运行，若与终端无关，则显示? 若为pts/0等，则表示由网络连接主机进程。
    CMD //命令的名称和参数

  7. 其中STAT状态位常见的状态字符有
    D //无法中断的休眠状态（通常 IO 的进程）；
    R //正在运行可中在队列中可过行的；
    S //处于休眠状态；
    T //停止或被追踪；
    W //进入内存交换 （从内核2.6开始无效）；
    X //死掉的进程 （基本很少见）；
    Z //僵尸进程；
    < //优先级高的进程
    N //优先级较低的进程
    L //有些页被锁进内存；
    s //进程的领导者（在它之下有子进程）；
    l //多线程，克隆线程（使用 CLONE_THREAD, 类似 NPTL pthreads）；
    + //位于后台的进程组；
