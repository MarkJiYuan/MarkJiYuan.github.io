Python/Python入门教程1

Python入门教程1

​ 这个系列的教程是面向初学者的，却不太同于通常教程的Python教程。我的着眼点不在于循规蹈矩地介绍各种数据类型，或者语法，而更多地在于我对Python的一些理解，并且更注意实践中的使用。因此我会介绍更多如何自己阅读文档，自己研究库的方法和心得，相应的行文也比较混合，不是那么有结构。建议结合系统的教程一起食用，效果更佳：菜鸟教程Python基础，廖雪峰Python入门。看教程的过程中出现不懂的不要紧，等水平高了以后再回来看，就迎刃而解了。

  1. 安装Python

    ​ 对于初学者而言，通常来说不建议安装某某软件最新的版本，最好安装上一个稳定的版本。Python的话，我建议使用3.7版本，小版本号无所谓，即3.7.x都可以。一定一定不要装Python2，已经被淘汰了。

    ​ Python的安装教程满地都是，这里就不赘述了。一定要学会使用搜索引擎。

    ​ 这里要提一下，不同的操作系统安装Python不太一样。注意的点

    • Mac系统自带Python2的，所以安装Python3的话，要正确处理跟Python2的关系。（千万别动系统原本的Python2）配置环境变量的时候可以将Python3配置为python3，以区别python调用的是python2
    • Windows需要手动配置环境变量。

  1. 打开Python shell

    ​ 目前可以将shell简单的理解为就是个运行代码的地方，每输入一行命令回车后，就会打印出运行的结果。打开shell的方式非常简单（如果上一步环境变量配置正确了的话），在终端中输入python即可。可以在头部看到使用的python的版本号。

    ​ 看到这个界面就说明安装正常，可以使用python了。以后可以把任意的python代码复制到这个黑不拉几的界面里，然后回车就可以运行并看到结果。当然也可以手敲。如果复制的方式不对，复制的内容有问题的话，shell也会很认真地报错的。在学习编程的过程中，最最不要害怕的就是报错。要耐下心来去读错误提示（专业术语叫异常栈）。其实编译器（可以理解成理解人写的程序并让CPU执行的软件）是程序员最好的老师。

中午：读操作系统第四章

  1. 线程比进程的优势：
    1. 响应。可以多个线程同时运行，即便其中一个无响应/耗时长，也不影响其他。
    2. 资源共享。进程需要通过IPC来通讯，而线程之间天生共享很多资源。
    3. 开销小。相比进程，无论是创建还是转换都要更快，更省。
    4. 可规模化。可以完全发挥多核CPU的能力。我的理解是研究更快的CPU有难度，但是扩展核的数量是比较容易的，相当于可以用规模化来提高整体的速度。
  2. 区分并发(concurrency)和并行(parallelism)
  3. 阿姆达尔定律 —— 增加CPU的边际收益递减
  4. 并行分为：数据(data)并行和任务(task)并行
  5. 线程分为：用户态线程，内核态线程
  6. 用户态线程和内核态线程的关系：
    • 多对一：无法真正地并行，现在已经没人用了
    • 一对一：真正并行，弊端是可能会创建太多线程，资源浪费以及创建开销。主流操作系统都用的这个。
    • 多对多：内核态线程 小于等于 用户态，可以自行设置数量和绑定。但是实现比较困难，而且随着CPU核数的增多越来越没什么意义。理论上挺美好的，融合前两者的优劣，结果是没什么特长。

Linux/ps命令

ps命令详解

  1. ps aux

    BSD风格

    a表示all，所有进程，无视用户。因为默认ps只打印自己的进程

    u打印详细信息

    x表示也列出没有terminal的进程，即守护进程daemon，它们的tty是??

    l为长信息，long message，有PPID这些

  2. 各种信息说明

    • USER: The effective user (the one whose access we are using)

    • PID: Process ID

    • %CPU: CPU time used divided by the time the process has been running

    • %MEM: Ratio of the process’s resident set size to the physical memory on the machine

    • VSZ: Virtual memory usage of the entire process

    • RSS: Resident set size, the non-swapped physical memory that a task has used

    • TTY: Controlling terminal associated with the process

    • STAT: Process status code

    • START: Start time of the process

    • TIME: Total CPU usage time

    • COMMAND: Name of executable/command

  3. RSS， VSZ， PSS

  4. top命令可以获取实时状态

  5. Linux下显示系统进程的命令ps，最常用的有ps -ef 和ps aux。这两个到底有什么区别呢？两者没太大差别，讨论这个问题，要追溯到Unix系统中的两种风格，System Ｖ风格和BSD 风格，ps aux最初用到BSD中，而ps -ef被用在System V Style中，两者输出略有不同。现在的大部分Linux系统都是可以同时使用这两种方式的。

  6. System V风格

    UID //用户ID、但输出的是用户名
    PID //进程的ID
    PPID //父进程ID
    C //进程占用CPU的百分比
    STIME //进程启动到现在的时间
    TTY //该进程在那个终端上运行，若与终端无关，则显示? 若为pts/0等，则表示由网络连接主机进程。
    CMD //命令的名称和参数

  7. 其中STAT状态位常见的状态字符有
    D //无法中断的休眠状态（通常 IO 的进程）；
    R //正在运行可中在队列中可过行的；
    S //处于休眠状态；
    T //停止或被追踪；
    W //进入内存交换 （从内核2.6开始无效）；
    X //死掉的进程 （基本很少见）；
    Z //僵尸进程；
    < //优先级高的进程
    N //优先级较低的进程
    L //有些页被锁进内存；
    s //进程的领导者（在它之下有子进程）；
    l //多线程，克隆线程（使用 CLONE_THREAD, 类似 NPTL pthreads）；
    + //位于后台的进程组；
